Back End

For the back end, we use the Django to connect the database with the front end. However, we actually didn’t transfer the data through Django’s models to the front end. We still use the pymongo module to get the data from the MongoDB. Although we can use packages like Djongo to link django and MongoDB, we have found that it’s still hard to represent some datatype in MongoDB to Django models, and also there are some problems with MongoDB automatic generated primary key “_id”. After all, we only need to show our data and analysis in the web page, there is no data need to get from the users.

Data Collection

The web crawler grabs the user posts we needed from the Sina Weibo website. The result was firstly stored in excel file which contains several sheets. Each sheet is named by the specific hour of when the data is posted. Then the pre-processing is applied on the raw data which contains combining sheets named by hours into one sheet named by day. Then we do the data cleaning to get rid of the noise information and the repeated data and store the result into MongoDB. We also get the infection information of the world and the infection information of China from the statistic collected by others. We download the data as a form of excel file and stored the data into MongoDB.

Here are some Python packages we used in data collection:
Web crawler: Selenium + BeautifulSoup
Database: MongoDB + pymongo
Data processing: re + pandas + xlrd + xlwt + emoji

Keyword Extraction

Keyword extraction is used for generating the word cloud in the China page. The data we used for generating word cloud is all the cleaned posts collected from Weibo. The model we used to perform keyword extraction is based on IDF(Inverted Document Frequency). For convenient, we just use a third party API, jieba, to perform this task, because they all have built-in functions. After generating the keywords for 5 periods: days before Wuhan locked down period, confirmed number incrementing period, confirmed number decreasing period, days before resuming work, and days after Wuhan lifting the closure. And those days are 12/1 - 1/22, 1/23 - 2/18, 2/19 - 3/11, 3/12 - 4/7, and 4/8 - 4/30. After getting the top 50 weighted keywords in all the posts(12/1 - 4/30), we extract the weights for each of these keywords in those 5 periods. We also translate those keywords into English using Python Google translate API. Then, we stored the results into MongoDB.

Sentiment Analysis

For sentiment analysis, the dataset for final prediction is the cleaned Weibo post, up to 300,000 posts. For the training part, we have trained nearly 20 models on 3 different datasets: D1, D2, D3. They are only different in the size, and they are all labeled into 3 class: positive(1), neutral(0), and negative(-1). D1 is up to 1500 posts, D2 is up to 2500 posts, D3 is up to 6000 posts. For selecting features, we use two kinds of features: TFIDF and LIWC. TFIDF stands for term frequency–inverse document frequency. LIWC (Linguistic Inquiry and Word Count) is a text analysis program, it can analysis the text and return the percentage of each category word in the text. 3 algorithms are applied to the dataset: Naive Bayse, Linear Regression and SVM(Support Vector Machine). We also use the SVD to reduce the dimension of TFIDF features under SVM models, and the results didn’t show any improvement in accuracy.

Finally, after comparing the accuracy results in those models, we choose the TFIDF feature with the Naive Bayse model, the accuracy is up to 84%, to train all the data in D3. Then, we use this trained model to do the prediction on all the cleaned posts. However, after plotting the curve of confirmed infection number in China with the prediction on posts, the result is not the one we expected.
